\n

  1. java.lang.ClassNotFoundException
    2. \n

解决方案

\n

所有模块使用一个父模块来管理
将重复配置迁移到 parent pom.xml 中;
使用 dependencyManagement 来统一管理 jar 依赖;
使用 pluginManagement 统一管理插件依赖;
所有 jar 依赖的版本全部使用 properties 管理;
使用 excludes 排除冲突的 jar 包;
